## 1. TL;DR 执行摘要

ai-bazi 是一个拥有两项市场独有差异化能力（典籍 RAG + 断前事校准）的 AI 八字排盘产品，但当前存在 **4 项一票否决安全漏洞**和**日期选择器不可见**两个上线阻塞问题。本路线图规划 4 个 Sprint、~45h 纯开发工作量、覆盖 17 项需求，在 **12 个自然日内达到正式上线标准**（Sprint 0 + Sprint 1），后续 4 周完成移动端体验和信息架构优化。最大风险点是 SMS 集成（16h），需在产品决策确认后立即启动服务商开户流程以避免阻塞关键路径。

## 2. 核心结论卡片

| 维度 | 结论 |
|------|------|
| **当前可否上线** | 否。4 项一票否决安全漏洞 + 日期选择器不可见 = 产品不可面向真实用户开放 |
| **最快上线时间** | 12 个自然日（Jul 20 → Jul 31），前提是 SMS 服务商审核不超 3 个工作日 |
| **关键路径瓶颈** | SMS 集成 16h（REQ-P1-01）。万能验证码删除和 dev 端点移除依赖 SMS 就绪 |
| **上线后首月目标** | 移动端按钮可见率 18.9% → ≥80%，文字溢出 15 → 0，安全评分 2/10 → 9/10 |
| **总工作量** | ~45h 纯开发（P0 3h + P1 18h + P2 24h），分 4 个 Sprint，约 6 周完成全部交付 |
| **最大风险** | SMS 模板审核延迟 > 3 个工作日（概率：中）。缓解方案：提前启动开户 + 准备 Mock 模式 |
| **不做的事** | 不重构排盘引擎、不做 App、不加多语言、不做付费体系（详见 PRD §6 Non-Goals） |

### 8.3 假设与约束
