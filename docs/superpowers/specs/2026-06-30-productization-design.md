## 一、产品定位

免费 AI 八字排盘工具。核心排盘、旺衰分析、典籍判断、断前事校正全部免费。高级功能（流月详批、AI 深度对话、合盘）积分制付费。

目标用户：对自己八字好奇的普通人，愿意花 5 分钟走完校准流程以获得准确报告。

## 三、积分体系

### 获取途径

| 途径 | 获得 | 限制 |
|---|---|---|
| 购买流月 | ¥9.9 → 100 积分 | 可重复购买 |
| 邀请好友 | 好友注册 + 完成校准 → 50 积分 | 无上限 |
| 连续签到 | 每天 2 积分 | 断签重计 |
| 积分补充包 | ¥6.9 → 100 积分 | 可重复购买 |

### 积分规则

- 积分不过期
- 积分只用于 AI 对话（不用于流月、合盘）
- 月限 50 积分可赠出（防止刷分）
- 首 3 轮 AI 对话免积分（让用户体验价值）

## 四、邀请裂变

### 流程

```
用户 A（已校准）
  └── 生成邀请链接/二维码
        └── 用户 B 点击 → 注册 + 排盘 + 完成校准
              ├── A 得 50 积分
              └── B 得到准确命盘报告（价值体验）
```

### 关键设计

- 被邀请者必须完成「断前事校准」才算有效邀请（防止刷分）
- 被邀请者自然走完免费全流程，已体验核心价值
- 邀请链接带邀请人 ID token
- 未登录用户点击邀请链接：token 存 sessionStorage，注册时自动绑定邀请关系
- 已登录用户点击：直接记录邀请关系（但不触发奖励，奖励仅在注册时触发）

## 五、命盘分享与合盘

### 分享

- 已校准用户可生成「命盘名片」分享链接
- 名片内容：日主 + 格局 + 用神 + 一段 AI 生成的文字画像
- 不暴露完整四柱（隐私 + 勾引对方自己排）
- 名片附带「我也要测」入口

### 合盘

- 双方各自完成校准
- 合盘发起者付费（谁发起谁付）
- 生成合盘报告后可分享给对方
- 合盘单独定价 ¥18，不含积分
- 场景入口：命盘名片页 → 「与 TA 合盘」

## 六、技术架构

### 部署

```
User → Nginx (SSL 终结 + 静态文件)
         └── FastAPI (uvicorn)
               └── SQLite (via SQLAlchemy ORM)
```

- 单台 VPS（阿里/腾讯轻量云，2vCPU 4G）
- Docker Compose 一键部署
- SQLAlchemy ORM 抽象数据库（后续可迁 PG）

### 新增模块

```
项目根目录/
├── api/
│   └── routes/
│       ├── auth.py          # 登录/注册/验证码
│       ├── payment.py       # 订单创建/支付回调
│       ├── points.py        # 积分查询/消耗/赠送
│       ├── invite.py        # 邀请链接/裂变追踪
│       └── chat.py          # AI 对话（含积分门控）
├── models/
│   ├── db.py                # SQLAlchemy engine + session
│   ├── user.py              # 用户表 + 验证码表
│   ├── entitlement.py       # 权限表（谁买了什么）
│   ├── order.py             # 订单表
│   ├── points.py            # 积分表 + 积分流水
│   └── invite.py            # 邀请关系表
├── services/
│   ├── gate.py              # 权限门控中间件
│   └── points.py            # 积分业务逻辑
├── main.py                  # 拆分路由注册，瘦身
└── public/
    └── index.html           # 前端（现有，加支付弹层/登录页/对话面板）
```

### 核心原则

- **不碰现有命理逻辑** —— 排盘、旺衰、断前事、校正等模块原封不动
- **通过装饰器/中间件做权限门控** —— 付费接口加 `@require_entitlement("liuyue")`
- **SQLAlchemy 统一数据访问** —— 禁止裸写 SQL，为后续切 PG 留路

## 七、支付流程

### 聚合支付方案

使用 xorpay（或其他聚合商），一接入同时支持微信 + 支付宝。个人身份可接入。

### 订单流程

```
前端                      后端                     聚合支付
  │                         │                          │
  ├─ 点击「解锁流月 ¥9.9」─→│                          │
  │                         ├─ 创建订单（pending）      │
  │                         ├─ 调用聚合支付创建支付链接  │
  │                         │──────────────→           │
  │                         │←── 支付链接 ─────────────│
  │←── 跳转支付页 ──────────│                          │
  │                         │                          │
  │  (用户在微信/支付宝完成支付)                         │
  │                         │                          │
  │                         │←── 支付回调 ─────────────│
  │                         ├─ 验签                     │
  │                         ├─ 订单 → paid              │
  │                         ├─ entitlements 加流月权限   │
  │                         ├─ points 加 100 积分        │
  │                         │                          │
  │←── 支付成功，刷新页面 ───│                          │
```

### 安全要求

- 支付回调必须验签（防伪造通知）
- 订单 30 分钟过期（防止积压）
- 幂等处理——同一订单多次回调不重复加积分

## 八、用户系统

### 注册/登录

- 手机号 + 短信验证码
- 验证码 5 分钟有效，同一号码 60 秒内不可重发
- 登录成功返回 JWT（7 天有效）
- 首次登录自动创建用户 + 赠送首 3 轮 AI 对话免费额度

### 鉴权

- JWT 放 Header：`Authorization: Bearer <token>`
- 现有免费接口不加鉴权（保持零门槛）
- 付费接口加 `@require_auth` + `@require_entitlement("feature")`
